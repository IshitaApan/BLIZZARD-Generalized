[PropertiesDialog] propertyPages extension point does not allow forcePluginActivation when using core.expression's property tester

+++ This bug was initially created as a clone of Bug #244754 +++ using eclipse 3.4 I have a plugin that defines a property tester using the org.eclipse.core.expressions.propertyTesters extension point. It also defines a properties page using org.eclipse.ui.propertyPages and it's set up like this: <enabledWhen> <adapt type="org.eclipse.core.resources.IProject" > <test property="org.eclipse.mypropertytester" forcePluginActivation="true"/> </adapt> </enabledWhen> The problem is the "forcePluginActivation="true"" is not forcing my plugin to activate. It looks like this is because RegistryPageContributor#failsEnablement does not set up the evaluation context to allow plugin activation. So this: return enablementExpression.evaluate( 	new EvaluationContext(null, object)).equals(EvaluationResult.FALSE); Should do this: IEvaluationContext context = new EvaluationContext(null, object); context.setAllowPluginActivation(true); return enablementExpression.evaluate(context).equals(EvaluationResult.FALSE); Without this fix, I would have to be forced to make my plugin start up all the time if I want to be able to use my property tester to enable property pages.
