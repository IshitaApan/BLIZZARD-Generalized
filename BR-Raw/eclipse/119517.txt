TVT3.1.x:TCT229: Directory/path/file name fields must be LTR

Wherever file name/paths are displayed, reading order must be LTR. Here's an example from the New Project dialog - you can see the project name is appended at the beginning of the path instead of the end because the Directory field has RTL reading order. I expect this pronlem was reported and maybe already fixed for Arabic?
