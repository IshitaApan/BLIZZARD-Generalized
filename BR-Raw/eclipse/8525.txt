The wizard will resize according to the text boxes layout but it won't take into account the size of the description string, if this is long enough it will need more than 2 lines which are allocated for it. The user of the WizardPage class can control only the size of the client area, but he cannot figure out how much space the description will take because he doesn't have access to the font that is used to display the description etc. so it's really a matter of guessing.
