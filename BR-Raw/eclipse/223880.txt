[KeyBindings] Command keybinding enabled even though action is disabled

While testing the All Instances action in debug (displays a list of all instances of the selected type in the vm), I found that I could run the action even when it was not supposed to be available. The action is available in the context menu of the java editor, outline view and variables view. It is also on the main 'run' menu. We have a keybinding Ctrl-Shift-N. All of the actions are disabled unless we are debugging with a 6.0 VM. While all the actions are disabled, hitting Ctrl-Shift-N is still running the action.
