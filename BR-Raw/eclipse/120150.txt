[Commands] Javadoc bugs  in ParameterizedCommand and ExecutionEvent

The constructor for ParameterizedCommand refers to "an array of quadruples (<code>String[]</code>)" for the 'parameterizations' parameter, however the type of that parameter is Parameterization[]. In ExecutionEvent there are several places where it says "parameter name" but I think it should say "parameter id". In the constructor: * @param parameters * The parameters to qualify the execution; must not be * <code>null</code>. This must be a map of parameter names (<code>String</code>) * to parameter values (<code>String</code>). And the method getParameter(final String parameterName) has the same problem. This analysis is based on the code in ParameterizedCommand.getParameterMap() that builds a Map of parameter id to parameter value which is passed to the ExecutionEvent constructor.
