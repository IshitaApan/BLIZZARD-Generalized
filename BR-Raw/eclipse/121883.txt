[Markers] org.eclipse.ui.tests plug-in makes lots of performance tests red

I200510xx-yyy00 and newer. NOTE: The real cause/problem for this is bug 121882 (releng test infrastructure installs all test plug-ins together). The huge performance degradation which can be observed many JDT Text performance tests: http://fullmoon.rtp.raleigh.ibm.com/downloads/drops/S-3.2M3-200511021600/performance/org.eclipse.jdt.text.php is caused by problemFilter extensions (org.eclipse.ui.ide.markerSupport) contributed by org.eclipse.ui.tests (details see below). This causes the Problems view to update the focus markers (see MarkerView.updateFocusMarkers(Object[]) upon every selection change. Due to bug 121882 this affects all Eclipse SDK performance tests that have the Problems view open (which is the default for the SDK) and involve selection and/or part activation. Here's the extension code that causes this: <extension point="org.eclipse.ui.ide.markerSupport"> <problemFilter enabled="true" id="org.eclipse.ui.tests.onAnyError" name="On Any Error" scope="ON_ANY" severity="ERROR"/> <problemFilter enabled="true" id="org.eclipse.ui.tests.onSelectedWarning" name="On Selected Warning" scope="ON_SELECTED_ONLY" severity="WARNING"/> <problemFilter enabled="true" id="org.eclipse.ui.tests.infoAndChildren" name="On Selected Info And Children" scope="ON_SELECTED_AND_CHILDREN" severity="INFO"/> <problemFilter enabled="true" id="org.eclipse.ui.tests.sameContainerNoSeverity" name="Same Container No Severity" scope="ON_ANY_IN_SAME_CONTAINER"/> <problemFilter enabled="true" id="org.eclipse.ui.tests.onMethod" name="On Method" onDescription="method"/> <problemFilter id="org.eclipse.ui.tests.notOnMethod" name="Not On Method" onDescription="!method"/> <problemFilter enabled="true" id="org.eclipse.ui.tests.onProblem" name="On MarkerType"> <selectedType markerId="org.eclipse.core.resources.problemmarker"/> </problemFilter> <subCategoryProvider 		markerType="org.eclipse.ui.tests.categoryTestMarker" class="org.eclipse.ui.tests.markers.TestCategoryProvider" /> </extension>
