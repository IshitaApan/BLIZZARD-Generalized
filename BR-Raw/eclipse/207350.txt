[Trim] View disappears when perspective is switched while expanded, minimized view is opened

Created attachment 81095 sample rcp application Build ID: M20070921-1145 Steps To Reproduce: 1. Start the attached RCP sample application. 2. Minimize the opened view. Use the left tab icon to expand the view temporarily. 3. Click the switch perspective button on the view. Perspective is switched, but the area where the view icon was is still there. 4. Switch back to the original perspective using the perspective bar. The view and the minimized view icon disappeared. (You can see that the tab with the icon is behind the editor area, see screenshot)
