Debugging against report w/ library-defined parameters fails

1) In a library define a parameter which: - string - has a default value - Required = yes - Hidden = yes 2) Create a new blank report. Note that debugging works. 3) Drag the parameter from step #1 onto the report from step #2. 3) Attempt to debug. Nothing at all happens in the IDE. Workspace log error generated (see below). 4) Edit the parameter on the report. Ex: double click on it, uncheck then recheck Required, click OK, save report. 5) Debug now works. Similar to: http://dev.eclipse.org/newslists/news.eclipse.birt/msg30728.html Note - Should someone add "debugger" to the bugzilla components list?
