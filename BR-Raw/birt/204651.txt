AssertionError for an image (byte[]) in a scripted data set column of type "any"

For the display of an image created dynamically by an own java class I use a scripted data set with one column of type "any" and a ScriptedDataSetEventAdapter that sets the column value with a byte[]. This was described in a BIRT example and works with BIRT 2.1.2. Now, we are using BIRT 2.2 and we get an AssertionError running our report creation JUnit test (where the assertion check is enabled). It seems that the constructor of the "ResultObject" class has got a new if-statement if the result field is of type "any". The assertion within the invoked method "ResultFieldMetadata.setDataType()" checks whether the data type is an Integer.class or a Double.class or ...
