Context root issue when set to other BIRT Web Project.[0603]

Description: Context root issue when set to other BIRT Web Project. Build number:2.2.0.v20061216 Steps to reproduce: 1. New two BIRT Web Project wtp1 and wtp2. 2. New a jsp file in wtp1 project. 3. New jsp file viewer.jsp based on (html,BIRT tag) template, input content like: <birt:viewer id="0" title="contextRoot" contextRoot="wtp2" reportDesign="birt_tag_sample.rptdesign"> </birt:viewer>. Note that birt_tag_sample.rptdesign only exists in wtp2 project. 4.New jsp file report.jsp based on (html,BIRT tag) template, input content like: <birt:report id="0" title="contextRoot" contextRoot="wtp2" reportDesign="birt_tag_sample.rptdesign"> </birt:viewer>. 5. Run the two jsp files on Server and select wtp1 and wtp2 as configured, input url as http://host:port/wtp1/viewer.jsp http://host:port/wtp1/report.jsp Expected result: Two jsp files can display report contents. Actual result: Viewer.jsp works well but report.jsp will throw errors. The error indicates that it looks for the report under wtp1 folder, if it's the case, why viewer can work? Error log: The report file : D:\wtp_workspace\.metadata\.plugins\org.eclipse.wst.server.core\tmp4\webapps\wtp1\birt_tag_sample.rptdesign does not exist or contains errors.
