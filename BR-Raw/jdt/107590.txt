[source action] 'Override method' content assist does not use parameter names from overridden method

I20050818-0800 'Override method' content assist does not use parameter names from the overridden method: import java.util.ArrayList; import java.util.List; import java.util.Set; public class A { List<Object> linearize(Set<Object> set) { return new ArrayList<Object>(set); } } class B extends A { //ContentAssist here, override linearize(..) } In the example, content assist inserts: @Override List<Object> linearize(Set<Object> arg0) { // TODO Auto-generated method stub return super.linearize(arg0); } Expected: uses 'set' instead of 'arg0'. A breakpoint in StubUtility2#suggestArgumentNames(..) reveals that IMethodBinding#getJavaElement() returns null. This may be be a consequence of the way the ASTParser is set up in OverrideCompletionProposal#updateReplacementString(). If I change it to ... 		parser.setSource(fCompilationUnit); ..., then it works in this case (but may invalidate the fix for bug 93249).
