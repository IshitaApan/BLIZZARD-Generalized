[5.0] Mark Occurrences should mark all declaration-equivalent types

I20050331-2000 Mark Occurrences should mark all erasure-equivalent types. In the example below, each of the occurrences of 'Cell' only marks itself. Expected: all occurrences of 'Cell' are marked together. This could be achieved by calling ITypeBinding#getTypeDeclaration(), IMethodBinding#getMethodDeclaration(), etc. on each binding. 'Search > Occurrences in File > Identifier' could then be beefed up with filters (see also bug 89947). public class Cells { 	static class Cell<T> {} 	 	void ls(Cell<String> arg) {} 	void li(Cell<Integer> arg) {} 	void ln(Cell<Number> arg) {} 	void lo(Cell<Object> arg) {} 		 	void raw(Cell arg) {} 	void wild(Cell<?> arg) {} 	void ext(Cell<? extends Number> arg) {} 	void sup(Cell<? super Number> arg) {} 	 	 	void potential(aa.Cell arg) {} }
