[hashcode/equals] Generated hashCode() and equals() wizard should use {} on if statements

The generate hashCode/equals wizard generates code like this: public class Test { 	private int foo; 	private int bar; 	public int hashCode() { 		final int prime = 31; 		int result = 1; 		result = prime * result + bar; 		result = prime * result + foo; 		return result; 	} 	public boolean equals(Object obj) { 		if (this == obj) 			return true; 		if (obj == null) 			return false; 		if (getClass() != obj.getClass()) 			return false; 		final Test other = (Test) obj; 		if (bar != other.bar) 			return false; 		if (foo != other.foo) 			return false; 		return true; 	} } According to http://java.sun.com/docs/codeconv/html/CodeConventions.doc6.html#449, it is a bad practice to avoid the {} on if statements.
