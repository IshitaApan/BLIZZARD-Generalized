[search] Search view flashing while navigating workspace

I20050401-1645 Steps to reproduce: 1. Perform a Java search 2. Switch the search view to "Flat layout" 3. Navigate through the package explorer, expanding sections not previously visited 4. Observe that as new paths are expanded, the search results view flashes and is very distracting This flashing is caused by calls to setRedraw() in JavaSearchResultPage in handleLabelProviderChanged. This code was added for bug 58817. I'm not sure what the right solution here is, but the flashing is really getting to me and so I think the calls to setRedraw() should be removed. Can a better solution to bug 58817 be found?
